# Security Policy

We take the security of UptimeKit very seriously. If you discover a security vulnerability, please bring it to our attention immediately.

## Supported Versions

As UptimeKit is a newly launched project, we currently support all released versions.

| Version | Supported          |
| ------- | ------------------ |
| All     | :white_check_mark: |

## Reporting a Vulnerability

If you have discovered a security vulnerability in UptimeKit, please do **not** open a public issue on GitHub. Instead, please report it privately:

1.  **Email**: Send a detailed report to [security@uptimekit.dev](mailto:security@uptimekit.dev).
2.  **Details**: Please include as much information as possible, including:
    - Steps to reproduce the vulnerability.
    - The potential impact.
    - Any proof-of-concept code or screenshots.

## Response Timeline

We are committed to addressing security vulnerabilities promptly:

-   **Acknowledgement**: We will acknowledge your report within 48 hours.
-   **Assessment**: We will assess the severity and impact of the issue.
-   **Fix**: If confirmed, we will work on a fix and release a security patch as soon as possible.
-   **Disclosure**: We will coordinate the public disclosure of the vulnerability with you effectively.

Thank you for helping keep UptimeKit secure!
